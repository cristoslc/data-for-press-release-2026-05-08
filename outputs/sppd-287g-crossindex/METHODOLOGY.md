## Goal

Identify agencies that appear in **both** datasets — i.e., agencies that share South Portland's Flock network AND hold an active 287(g) agreement with ICE.

Only rows in the SPPD shared-network spreadsheet where `I'm Sharing` is non-null and non-empty are included. This restricts the analysis to agencies SPPD is **actively sharing data with**, excluding rows where SPPD only receives data from another agency.

## Normalisation Steps

### 1. State extraction

Agency names often include a state abbreviation or full state name (e.g., "Lake City FL PD", "Massachusetts Department of Correction"). The normaliser matches both full state names and two-letter abbreviations, converting all to a canonical 2-letter code (e.g., `fl`, `ma`). This is used as a hard constraint — two agencies can only match if they share the same state.

Non-state two-letter tokens that appear in agency names — `PD` (Police Department), `SO` (Sheriff's Office), `IB` (IB/Flex suffix), `LR` (LR suffix) — are explicitly excluded from state extraction.

### 2. State suffix removal

After extracting the state, all remaining state code tokens are stripped from the agency name before matching — not just trailing state codes, but state codes anywhere in the token set (since they've been captured and stored separately).

### 3. Agency name normalisation

Common suffixes (e.g., `Police Department`, `Sheriff's Office`, `County Sheriff`, `PD`, `SO`, `Commission`, `Department`, `Authority`, `Detectives`, `Garage`, `Booking`, `Intake`, `Constable`, `Precinct`, `Task Force`, `Warrant Service Officer`, `Jail Enforcement`, `Office`) and punctuation are removed. State codes, vendor/deployment tags (`LR`, `Flex`, `Condor`), and non-agency tokens are stripped. The name is lowercased and collapsed to a canonical token set.

Note: `IB` (Intake/Booking) is NOT stripped as a suffix because it is a facility/deployment tag, not an agency-type indicator. It only appears in one SPPD entry ("Hillsborough County Pierce Garage IB") which is a jail sub-unit, not a 287(g) agency. The union Jaccard threshold naturally prevents this entry from matching.

## Matching Logic

An agency `A` (from SPPD's network) and agency `B` (from ICE 287(g) list) are considered the **same agency** only if:

1. **State match**: Both have the same state code (after normalisation). If the SPPD agency has **no extractable state**, only exact normalised name matches are allowed (no fuzzy Jaccard matches).
2. **Name similarity**: One of the following is true:
   - Exact normalised name match (after removing type suffixes, state codes, and punctuation), OR
   - Jaccard word-overlap score ≥ 0.75 using the **union** denominator (intersection over union of word sets).

No geospatial proximity, phone number, or address matching is used.

### Jaccard scoring (union denominator)

The Jaccard similarity is computed as `|A ∩ B| / |A ∪ B|` — the number of shared words divided by the total unique words across both names. This is the standard Jaccard definition.

The previous version used `min(|A|, |B|)` as the denominator, which allowed short names that were subsets of long names to score 1.0 (e.g., "pennsylvania state" ⊂ "pennsylvania state constable office honey brook precinct 1" → 2/2 = 1.0). The union denominator corrects this: the same pair now scores 2/8 = 0.25, properly below the 0.75 threshold.

### Stateless-match restriction

When an SPPD agency name has no extractable state code (e.g., agencies like "CSX Railroad" or "MAGLOCLEN RISS"), fuzzy Jaccard matches are rejected — only exact normalised name matches are allowed. This prevents short-name hits against ICE agencies in any state.

### Best-match selection

For each SPPD agency, the script evaluates all ICE agencies in the same state and retains the **best** match:

1. Higher confidence score wins.
2. Among equal scores, exact name matches beat fuzzy matches.
3. Among equal scores and same match type, shorter normalised name wins (more specific).

### Manual mappings

Some agencies are known matches that fall below the automated Jaccard threshold (0.75). These are recorded in `manual_mappings.csv` with a `verdict` column (`accepted` or `rejected`) and a note explaining why.

**Accepted manual mappings:**

| SPPD name | ICE 287(g) agency | Reason |
|-----------|-------------------|--------|
| Fish and Wildlife Commission FL PD | Florida Fish & Wildlife Conservation Commission | SPPD omits "Florida" and "Conservation"; Jaccard 0.60 |

**Rejected mappings (known false positives):**

| SPPD name | ICE 287(g) agency | Reason |
|-----------|-------------------|--------|
| Pennsylvania State PD | Pennsylvania State Constable's Office, Cooke Township | PA State Police ≠ Cooke Township constable; short-prefix subset hit |
| Bradford County Detectives PA | Bradford County Sheriff's Office (PA) | Detectives is a sub-unit within the SO, not the SO itself |

## Confidence Scoring

Every matched pair receives a confidence score and method:

| Score | Tier | Meaning |
|-------|------|---------|
| 1.0 | exact | Normalised names match exactly after stripping type suffixes, state codes, and punctuation |
| 0.75–0.99 | fuzzy | Jaccard word-overlap ≥ 0.75 (union denominator) |
| < 0.75 | manual | Entry from manual_mappings.csv with verdict=accepted; automated Jaccard too low |

The `match_method` column is `automated` or `manual`.

### Duplicate flag

The `duplicate_flag` column is `True` when multiple SPPD agencies map to the same (ICE agency, state) pair. This is informational — it indicates the same ICE record is matched by more than one SPPD network entry, which is common for vendor/deployment variants (Flex, LR, etc.). A `False` value means the ICE record is matched by exactly one SPPD agency.

## Known Limitations

- The SPPD shared-network spreadsheet uses varying naming conventions (e.g., "High Springs FL PD Flex" and "High Springs FL PD" for the same agency). The script deduplicates by agency name before matching.
- The ICE 287(g) list uses full legal names; SPPD uses shorthand. The union-Jaccard threshold (0.75) is deliberately conservative to avoid false positives.
- This is a name-matching exercise only. **No independent verification** of 287(g) status has been performed beyond comparing names against the ICE official list.
- Agencies that share network with SPPD but are not in the ICE list may still have 287(g) agreements not reflected in the official list.
- Sub-units within sheriff's offices (e.g., "Bradford County Detectives") may still match their parent SO after suffix stripping. The manual rejection list handles known cases, but others may exist.
